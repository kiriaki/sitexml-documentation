Element: SERA_quakeml

Name SERA_quakeml
Type RootType
Documentation
The SERA_quakeML schema has been created to facilitate the exchange of site characterization information. It is based on the QuakeML 2.0 Draft and complies with the requirements set by the European Geotechnical Database (EGD) and by EU-SERA.
XML Instance Representation
<SERA_quakeml
publicID="ResourceIdentifier [1] ?"
schemaVersion="SchemaVersionType [1] ?">
<!--
Uniqueness Constraint - uniquePublicID
Selector - .//*
Field(s) - @publicID
-->
<!--
Key Constraint - siteDescriptionPublicIDKey
Selector - ssx:siteDescription
Field(s) - @publicID
-->
<!--
Key Constraint - analysisPublicIDKey
Selector - ssx:analysis
Field(s) - @publicID
-->
<!--
Key Constraint - velocityProfilePublicIDKey
Selector - ssx:analysis/ssx:velocityProfileSet/ssx:velocityProfile
Field(s) - @publicID
-->
<!--
Key Reference Constraint - analysisSiteDescriptionRef
Selector - ssx:analysis
Field(s) - ssx:siteDescriptionID
Refers to - siteDescriptionPublicIDKey
-->
<!--
Key Reference Constraint - preferredAnalysisRef
Selector - ssx:siteDescription
Field(s) - ssx:preferredSiteAnalysisID
Refers to - analysisPublicIDKey
-->
<!--
Key Reference Constraint - preferredVelocityProfileRef
Selector - ssx:siteDescription
Field(s) - ssx:preferredVelocityProfileID
Refers to - velocityProfilePublicIDKey
-->

<creationTime> xs:dateTime </creationTime> [1] ?

Documentation

Example: quakeml:domain.ab/site/001.

Name Type Required Description

Example: 2025-11-26T12:49:55Z
revisionHistory RevisionHistoryType optional Structured update history for this SiteXML document. It records document-level provenance.
externalReference ExternalReferenceType optional , many URI of any type of external resource that may include site characterization information on this site. May occur more than one time.

siteDescription SiteDescriptionType required

Complex Type: RevisionType

Documentation
One document revision entry. The entry should summarize what changed in the SiteXML document, when the revision was made, optionally (but strongly recommented) who made it and where the previous version can be found.
Note: Document Revision metadata are not included in the QuakeML 2.0 Draft

Elements of <RevisionType>

Name Type Required Description
revisionTime xs:dateTime required Date and time of this document revision.
Example: 2026-05-02T12:00:00Z
description xs:string required Human-readable summary of the document update.
Example: Updated velocity profile and quality indexes.
author xs:string optional Person, institution, or system responsible for this document revision.
Example: ORFEUS
version xs:string optional Optional version label assigned by the publisher.
Example: 2026-05-02
previousVersion xs:anyURI optional URI of the previous published SiteXML document version, when available.
Example: https://example.org/sitexml/Site_XX.ABCD_01-05-2026.xml

<previousVersion> xs:anyURI </previousVersion> [0..1] ?
</...>

Name Type Required Description
station StationCodeType optional FDSN network and station code in network.station notation. Fill this if site metadata refers to a station. Otherwise leave empty.
Example: XX.ABCD
latitude LatitudeType required Geographic latitude (+/- for northern/ southern hemisphere, respectively).
Value range (-90, 90).
Unit (degress).
Example: 34.1567
longitude LongitudeType required Geographic longitude from Greenwich (+/- for eastern/western longitude, respectively). Value range (-180, 180).
Unit (degress).
Example: 28.1567
altitude xs:double optional Elevation of ground with respect to sea level (+/- for above/below sea level, respectively).
Unit (m).
Example: 130.80
minDistanceFromStation NonNegativeDouble optional Minimum distance between the permanent seismological station and the site characterization measurement. Should be used only when representative latitude and longitude of site characterization measurements cannot be provided.
Unit (m).
Example: 520.20
maxDistanceFromStation NonNegativeDouble optional Maximum distance between the permanent seismological station and the site characterization measurement. Should be used only when representative latitude and longitude of site characterization measurements cannot be provided.
Unit (m).
Example: 520.20
siteTopography SiteTopographyType optional SERA topographic/terrain classification according to one or both supported classification schemes. This is kept as a sibling of siteMorphology to distinguish formal topographic classification from the QuakeML-STC-derived site morphology indicator group.
siteMorphology SiteMorphologyType optional Site morphology and near-surface site indicators, including qualitative morphology, EC8 class, bedrock depth, H800, and geological unit.
preferredSiteAnalysisID ResourceIdentifier optional If you provide one or more analysis for this site you should use this field to designate the prefered analysis.
Note: The Overall Quality index of the Site Characterization will be calculated using the preferred analysis.
Example: quakeml:domain.ab/analysis/001

Note: The Overall Quality index of the Site Characterization will be calculated using the Velocity Profile Survey quality index associated with the preferred analysis.
Example: quakeml:domain.ab/velocity_profile/001
overallQindex QIndexType optional The Overall Quality index for this site. Must be a decimal value between 0 an 1.
Note: For a description of the Overall Quality Index of a site and guidelines for it's calculation refer to SERA Deliverable 7.2 - Best practice and quality assessment guidelines for site characterization.
Example: 0.8

For more information, see: SERA Deliverable 7.2, pg. 31.

Complex Type: SiteMorphologyType

Documentation
Group of site morphology and near-surface site indicators, as suggested by SERA.
Note: SiteMorphologyType is based on QuakeML 2.0 Draft stc:SiteMorphology
These include:
  • Qualitative Morphology
  • EC8 class site indicator
  • Seismological Bedrock Depth site indicator
  • Engineering Bedrock Depth (H800) site indicator
  • Geological Unit site indicator
For each site indicator network operators can provide at least the following information. For the allowed values in each case check the respective type.
  • Value
  • Reference
  • Quality Index

Elements of <SiteMorphologyType>

Name Type Required Description
morphology MorphologyType optional Qualitative description of the shape of the earth's surface. Takes specific values.
Example: Slope
siteClassEC8 EC8IndicatorType optional Ground type according to Eurocode 8 (EC8 § 3.1.2, Table 3.1), based on the velocityS30 value and the geotechnical description.
bedrockDepth BedrockDepthIndicatorType optional Seismological Bedrock Depth.
h800 H800IndicatorType optional Engineering Bedrock Depth.
geologicalUnit GeologicalUnitIndicatorType optional Free text description of the surface geology of a site.
